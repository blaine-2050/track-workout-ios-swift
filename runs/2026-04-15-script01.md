## Observations

- **Stop-summary regression (or never-implemented?).** PRD ([L39](https://github.com/blaine-2050/track-workout-core/blob/main/PRD.md)) and this script both assume Stop triggers a summary. iOS behavior here just resets state silently. Either the feature was never implemented on iOS despite the PRD claim, or it regressed. This is also relevant to [script 03](https://github.com/blaine-2050/track-workout-core/blob/main/WORKOUT_SCRIPTS/03-workout-stop-summary.md) which will also fail.
- **Stale-timer bug re-confirmed pre-run.** Before the wipe, I captured a hierarchy showing `Elapsed: 571:40` — exactly the walltime gap since last night's session (~9.5h). This corroborates the Step 0 smoke-test finding: elapsed calculations don't clamp to workout-active time.
- **"C" vs Backspace button.** In weight-entry mode the bottom-left key is `.` (decimal), in reps-entry mode it's `C` (clear). Consistent with the idea that decimals are weight-only, but the clear-vs-backspace affordance is not obvious. Minor UX note.
- **Sync-failed banner visible after Log.** Expected (no sync server running). Styling is small+gray; fine for now.
- **Maestro first-screenshot timing.** Workaround options: add a small `- waitForAnimationToEnd` or `- extendedWaitUntil` before the first screenshot, or screenshot after the first `assertVisible` succeeds.
